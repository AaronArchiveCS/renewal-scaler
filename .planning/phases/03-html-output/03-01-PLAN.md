**Step 9: Generate HTML Document**

After the RENEWAL OPTIONS text output from Step 8, generate a complete HTML file. Instruct Claude to build an HTML string using all the data already collected, then write it to disk and open it.

**HTML Structure:**

The document should contain these sections in order:

1. **Header** — Company name, customer tier, date generated, shop ID(s)

2. **Warning Banners** (if any warnings from Step 4) — Yellow/amber banner blocks at the TOP, before any data. Each warning gets its own banner line. If no warnings, omit this section entirely.

3. **Current State Baseline** — What the client pays now and how they use it:
   - Current plan name and monthly/annual price
   - UGC usage: used / limit with utilization percentage
   - Active seats count
   - Workspace count

4. **Utilization Insight** — A single highlighted callout box with one of these classifications:
   - **Overpaying**: utilization < 40% — "This customer is significantly underutilizing their plan. They're paying for capacity they don't use."
   - **Well-matched**: utilization 40-85% — "Usage aligns well with the current plan. Room to grow without immediate upgrade pressure."
   - **Ceiling risk**: utilization > 85% — "This customer is approaching or exceeding their plan limits. Proactive upgrade conversation recommended."
   - Color-code: green for well-matched, amber for overpaying, red for ceiling risk

5. **Optimal Plan Calculation** — Show the comparison table from Step 7:
   - All 3 plan options with add-on packs and total monthly cost
   - Highlight the optimal (cheapest) row
   - If high-usage warning (>15,000 UGC), show it here

6. **Renewal Option Cards** — One card per option from Step 8:
   - Each card shows: plan name, base price (with discount note), add-on packs and price, total monthly, total annual/contract, savings vs current
   - Use a clean card layout with clear visual hierarchy
   - If already-optimal, show the single 2-year card with the "already optimal" message

7. **Footer** — "Generated by /scale on [date] | Internal use only — not for client distribution"

**Styling requirements:**

- All CSS must be inline in a `<style>` tag — no external stylesheets
- Clean, professional look — think Linear or Notion, not flashy
- Light background, dark text, good contrast
- Cards with subtle borders and slight shadows
- Responsive-ish (looks good at ~800px width for screenshots)
- Print media query: hide unnecessary chrome, ensure cards don't split across pages
- Font: system font stack (-apple-system, BlinkMacSystemFont, 'Segoe UI', etc.)
- Color palette: slate grays for structure, green/amber/red for insight callout only

**File output:**

After generating the HTML string, instruct Claude to:

1. Sanitize the company name for filename (replace spaces with hyphens, lowercase, remove special chars)
2. Generate filename: `scale_[sanitized_company]_[YYYY-MM-DD].html`
3. Write the file to `~/Downloads/[filename]` using the Write tool
4. Open it in the default browser: `open ~/Downloads/[filename]` (macOS)
5. Tell the user: **"HTML report saved to ~/Downloads/[filename] and opened in your browser."**

- The HTML generation instructions should be clear enough that Claude can build the HTML string from the data it already has in context from the prior steps
