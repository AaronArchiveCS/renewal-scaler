Create the file `~/.claude/commands/scale.md` with the following exact structure and content:

**Header:**
```
# Scale Renewal Tool

Resolve customer identity and return a validated data profile for renewal planning.
```

**## Input section:**
- Capture `$ARGUMENTS`
- Detection rule (per D-18 — no flags): If `$ARGUMENTS` is purely numeric OR matches pattern `shop_NNNNN` or `shop-NNNNN`, treat as shop ID. Otherwise treat as company name.
- If no argument provided, respond: "Please provide a company name or shop ID. Usage: /scale acme corp OR /scale 12345"

**## Execution Steps:**

**### Step 1: Identity Resolution (HubSpot)**

Two branches based on input type:

Branch A — Company name search (per D-01):
- Use `mcp__claude_ai_HubSpot__search_crm_objects` with `objectType: "companies"`, `query: "<name from $ARGUMENTS>"`, `properties: ["name", "shop_id", "customer_tier", "lifecyclestage", "arr__active_deals_"]`
- Result handling (per D-02):
  - 0 results: Respond "No company found matching '[name]'. Check the spelling or try a shop ID: /scale <shop_id>"
  - 1 result: Proceed directly to Step 2 with this company
  - 2-5 results: Show a numbered list with format:
    ```
    Found [N] companies matching "[name]":
    1. [Company Name] — shop_id: [id] — Tier: [tier]
    2. [Company Name] — shop_id: [id] — Tier: [tier]
    ...
    Which company? Enter a number.
    ```
    Wait for CSM response. Use the selected company for Step 2.
  - More than 5 results: Respond "Too many matches for '[name]' ([N] found). Please re-run with a shop ID: /scale <shop_id>"

Branch B — Shop ID lookup (per D-01, D-03):
- Use `mcp__claude_ai_HubSpot__search_crm_objects` with `objectType: "companies"`, `filterGroups: [{ filters: [{ propertyName: "shop_id", operator: "EQ", value: "<shop_id>" }] }]`, `properties: ["name", "shop_id", "customer_tier", "lifecyclestage", "arr__active_deals_"]`
- 0 results: Respond "No company found with shop ID '[id]'. Verify the ID and try again."
- 1+ results: Show confirmation (per D-03): "Found: [Company Name] (shop_id: [id], Tier: [tier]). Is this correct?" Wait for CSM confirmation before proceeding.

**### Step 2: Workspace Scope (per D-04, D-09, D-10, D-11)**

After company is confirmed:
- Extract the `shop_id` value from the selected HubSpot company record
- IMPORTANT: The `shop_id` field may contain multiple IDs (per D-10). If the value contains commas, semicolons, or newlines, split on that delimiter to get all shop IDs. If it is a single value, treat as a list of one.
- Show the CSM what was found:
  ```
  Workspaces found for [Company Name]: [list of shop IDs]
  Does this customer have other workspaces we should include? Enter additional shop IDs separated by commas, or type "no" to continue.
  ```
- Wait for CSM response. If they provide additional IDs, add them to the list. If "no" or similar, proceed with the extracted IDs.
- Store the final list of shop IDs for use in Step 3 (data fetch — to be added in Plan 01-02).
