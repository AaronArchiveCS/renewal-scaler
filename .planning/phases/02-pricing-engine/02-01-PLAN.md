**Step 6: Pricing Reference Data**

Embed the following pricing table as a markdown reference block inside the skill file. This is NOT a code block to execute -- it is reference data the skill reads when calculating pricing.

Per D-01, the three standard plans:

| Plan | Monthly | Annual (10% off) | 2-Year (20% off) | UGC Limit | Credits |
|------|---------|-------------------|-------------------|-----------|---------|
| Startup | $500/mo | $5,400/yr ($450/mo) | $9,600/yr ($400/mo) | 500/mo | 20,000/mo |
| Growth | $1,500/mo | $16,200/yr ($1,350/mo) | $28,800/yr ($1,200/mo) | 2,500/mo | 70,000/mo |
| Enterprise | $5,000/mo | $54,000/yr ($4,500/mo) | $96,000/yr ($4,000/mo) | 10,000/mo | 150,000/mo |

Per D-02, UGC Add-on: +500 UGC/mo for $250/mo. Stackable. Same discount rates apply (10% annual, 20% 2-year).

Per D-03, credit add-ons and extra competitors are NOT in scope -- do not reference them.

Include a note in the skill file: "This pricing table is the source of truth for renewal calculations. If Archive's pricing changes, update this table."

**Step 7: Calculate Optimal Plan + Add-on Combination**

Add instructions for the optimization algorithm. The skill should instruct Claude to perform the following calculation using the customer's UGC usage from Step 5's output:

1. Read the customer's current UGC usage (`pricing_ugc_used` from the CUSTOMER DATA PROFILE in Step 5). Per D-10, match actual usage exactly -- no headroom buffer. (PRICE-05 from REQUIREMENTS specified headroom, but the user explicitly decided against it in D-10.)

2. For each of the 3 base plans (Startup, Growth, Enterprise):
   - Calculate the UGC gap: `customer_ugc_used - plan_ugc_limit`
   - If gap <= 0, no add-on packs needed (plan covers usage). Packs = 0.
   - If gap > 0, calculate packs needed: `ceil(gap / 500)` (each pack adds 500 UGC, per D-02)
   - Calculate monthly cost: `plan_monthly_price + (packs * $250)`
   - Store: { plan_name, plan_monthly, ugc_limit, packs_needed, addon_monthly, total_monthly }

3. Select the combination with the lowest `total_monthly`. This is the optimal combo.

4. Per D-12: If `pricing_ugc_used` is 0 or null (valid data, not missing), recommend Startup with 0 add-on packs.

5. Per D-13: If `pricing_ugc_used` exceeds 15,000, still calculate the optimal combo but append a warning:
   > NOTE: This customer uses [N] UGC/mo, which requires [X] add-on packs on top of Enterprise. Custom pricing may be more appropriate -- consult with leadership before presenting these options.

6. Tell the user: **"Optimal plan calculated: [Plan Name] + [N] UGC add-on pack(s) at $[total_monthly]/mo."**

7. Store the optimal combo result for use in Step 8 (Plan 02 will add Step 8).

After writing to .claude/commands/scale.md (repo copy), also copy the file to ~/.claude/commands/scale.md so the live skill is updated.
